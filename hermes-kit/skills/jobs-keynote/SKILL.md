# 🎤 Jobs Keynote — Product Launch Protocol

## When to Use
- Launching a new feature or product
- Creating a demo script
- Crafting a product narrative
- Preparing pitch material

## Procedure

### 1. Activate Jobs
`/personality jobs`

### 2. The Setup (Problem)
Start with the PROBLEM. Make the audience feel it.
- "Here's what sucks about [current solution]..."
- Paint the pain. Make them nod.

### 3. The Solution (Product)
NOW reveal your solution.
- Show it, don't describe it
- Demo > Slides > Words
- One feature at a time, build momentum

### 4. The Story Arc
```
1. Problem (pain) → 2. Existing Solutions (inadequate) → 3. Our Approach (insight) →
4. Demo (proof) → 5. Impact (transformation) → 6. One More Thing (surprise)
```

### 5. One More Thing
The feature or detail nobody expected.
- Save the best for last
- Make it feel like a bonus, not a feature list item

### 6. Output
```markdown
## [Product/Feature] — Launch Narrative

### The Problem
[2-3 sentences painting the pain]

### Why Now
[Why this matters today]

### The Solution
[1 sentence. If you need more, simplify.]

### Demo Script
1. [Step: what to show + what to say]
2. ...

### One More Thing
[The surprise element]

### Key Metrics
[What success looks like]
```

## Pitfalls
- Don't lead with features — lead with problems
- Don't show everything — curate ruthlessly
- The demo IS the product — if it's not demo-ready, it's not ready
