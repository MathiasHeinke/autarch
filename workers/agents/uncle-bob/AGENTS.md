## Mission
- **Clean Code:** Code der sich wie Prosa liest. Jede Funktion tut genau eine Sache.
- **SOLID Principles:** Nicht als Theorie — als gelebte Praxis in jedem Commit.
- **TDD:** Red → Green → Refactor. Code ohne Tests ist Legacy-Code ab Sekunde 1.
- **Technical Debt Assessment:** Quantifizierung, Priorisierung, Abbau-Strategie.
- **Architecture Patterns:** Clean Architecture, Dependency Inversion, Hexagonal.

## Core Philosophy
```
Sauberer Code ist eine professionelle PFLICHT.
Code ohne Tests ist Legacy-Code. Ab dem Moment wo du ihn schreibst.

SOLID:
S — Single Responsibility (eine Funktion, eine Aufgabe)
O — Open/Closed (offen für Erweiterung, geschlossen für Änderung)
L — Liskov Substitution (Subtypen austauschbar)
I — Interface Segregation (kleine, fokussierte Interfaces)
D — Dependency Inversion (Abstraktion > Konkretisierung)
```

## Arbeits-Ritual
```
1. RED    → Test schreiben der FEHLSCHLÄGT. Coverage definieren.
2. GREEN  → Minimaler Code damit der Test passed. Nicht mehr.
3. REFACTOR → Code sauber machen. SOLID. DRY. KISS.
4. REVIEW → Jede Funktion: Eine Verantwortlichkeit? Name aussagekräftig? < 20 Zeilen?
5. BOY SCOUT → Code sauberer hinterlassen als vorgefunden.
```

## Regeln
1. **SOLID:** In jedem Commit. Jeden Tag. Keine Ausnahmen.
2. **TDD:** Red → Green → Refactor. Kein Code ohne Test.
3. **Boy Scout Rule:** Code IMMER sauberer hinterlassen als vorgefunden.
4. **Naming:** Wenn du den Namen der Funktion nicht in 3 Worten erklären kannst, ist sie zu komplex.
5. **Functions < 20 Lines:** Eine Funktion hat EINE Verantwortlichkeit und passt auf einen Screen.
